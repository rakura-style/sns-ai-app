# プライバシーポリシーに追加する文言

## X API / Facebook API 認証情報の取り扱いについて

らくらスタイルが提供するSNS投稿サポートAIサービス（以下「本サービス」）では、お客様がX（旧Twitter）およびFacebookへの投稿機能を利用される際に、以下の認証情報を取得・保存します。

### 取得する情報
- X API認証情報（API Key、API Key Secret、Access Token、Access Token Secret）
- Facebook API認証情報（App ID、Access Token）

### 保存方法と安全管理措置
1. **保存場所**  
   上記の認証情報は、Google Firebase Firestoreのデータベースに保存されます。

2. **アクセス制御**  
   お客様の認証情報は、Firebase Security Rulesにより、**お客様ご自身のアカウントでログインしている場合にのみアクセス可能**となるよう厳格に管理されています。他のユーザーがお客様の認証情報にアクセスすることは技術的に不可能です。

3. **暗号化**  
   Firebaseとの通信は全てSSL/TLS暗号化されています。データベース内での保存においても、Firebaseの標準的なセキュリティ機能により保護されています。

4. **当方によるアクセス**  
   当方（サービス提供者）は、お客様の認証情報に理論上アクセス可能ですが、以下の場合を除きアクセスすることはありません：
   - お客様からの明示的な依頼がある場合
   - 法令に基づく開示要請がある場合
   - 本サービスの技術的な不具合の調査のために必要な場合

5. **第三者提供**  
   お客様の認証情報を、お客様の同意なく第三者に提供することはありません。

6. **削除**  
   お客様がアカウントを削除された場合、または認証情報の削除をご依頼いただいた場合、速やかに削除いたします。

### 認証情報の利用目的
お客様が入力された認証情報は、以下の目的でのみ使用されます：
- XおよびFacebookへの投稿機能の提供
- 予約投稿機能の提供

### お客様へのお願い
1. API認証情報は、第三者に知られないよう厳重に管理してください
2. 認証情報が漏洩した可能性がある場合は、速やかにX Developer Portal / Facebook開発者ポータルにて認証情報を無効化してください

## 補足説明（お客様向け）

Q: 当社（サービス提供者）は認証情報を見られますか？  
A: 技術的には閲覧可能ですが、通常業務において閲覧することはありません。Firebase Security Rulesにより他のユーザーからは完全に保護されています。

Q: 認証情報はどこに保存されていますか？  
A: Google Firebase（米国Google社が提供するクラウドサービス）のFirestoreデータベースに保存されます。

Q: もっと強固なセキュリティが必要な場合は？  
A: ご要望に応じて、お客様専用の暗号化キーによる追加の暗号化機能の提供も検討可能です。お問い合わせください。
